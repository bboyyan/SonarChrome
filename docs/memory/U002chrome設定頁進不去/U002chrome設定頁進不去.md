# U002 Chrome 擴充功能設定頁面無法開啟

## 🛠 問題描述
在擴充功能選單中點擊「⚙️ 開啟設定 (Open Settings)」後，出現以下狀況之一：
1. 畫面完全空白。
2. 顯示「Loading Settings...」字樣後就沒有反應。
3. 或是控制台報錯顯示 `effect_orphan` 錯誤。

## 🔍 原因分析（白話文版）
這個問題主要是因為原本我們用來蓋房子的工具（Svelte 框架）從舊版升級到了新版（Svelte 5），導致了一些「新舊不相容」的狀況：

1. **引擎啟動方式不對**：新版的系統就像是換了電子點火系統，但我們還在使用舊的轉動鑰匙方式（語法差異），導致引擎（頁面）發動不起來。
2. **語法格式衝突**：一些讓頁面內容「連動更新」的寫法，新版系統已經不認識了，導致執行到一半就因為看不懂而當機。
3. **開啟路徑失效**：在某些 Chrome 版本中，原本用來開啟設定頁面的捷徑指令突然失效，需要手動指定完整路徑才能進入。

## ✅ 解決方案
我們進行了以下修正，確保設定頁面能穩定開啟：

1. **更新啟動指令**：將頁面的掛載方式更新為新版指定的標準格式（從 `new` 改為 `mount` 函數）。
2. **規範連動寫法**：將原本會報錯的特定寫法改寫為標準函數，解決了系統執行時的邏輯混亂。
3. **實作「雙重開啟」機制**：
   - 第一層：嘗試使用 Chrome 標準指令開啟設定。
   - 第二層（保險機制）：如果第一種方式失敗，系統會自動轉換為「直接開啟新分頁」來呈現設定頁面。
4. **路徑自動修正**：優化了網頁資源的路徑指向，避免因為路徑找不到而導致頁面空白。

## 🧪 驗證與操作步驟
如果您再次遇到此問題，請按照以下步驟執行：

1. **重新載入**：前往 `chrome://extensions/` 頁面，點擊擴充功能的「重新載入」旋轉箭頭。
2. **手動測試**：再次點擊工具列圖示，並按下「開啟設定」。
3. **強制開啟**：如果還是進不去，可以直接在網址列輸入下方網址嘗試進入（其中的代碼視您的擴充功能 ID 而定）：
   `chrome-extension://您的擴充功能ID/src/options/options.html`
